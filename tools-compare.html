03 选型维度分析

从模型灵活性、成本控制、自主度三大维度深入分析，帮助你根据自身场景做出最优选择。

模型锁定 vs 灵活性

Claude Code 深度绑定 Claude，功能最先获得，但无法切换模型
OpenCode 任意模型提供商，最大灵活性，可用 Claude / GPT / Gemini / 本地模型
GLM Plan 绑定 GLM 系列，国内访问友好
Cursor 支持多模型切换，但部分高级功能仅限特定模型

价格与成本

Claude Code Pro $20/月, Max $100-200/月 (包含 Claude 用量)
OpenCode 工具免费, 按模型 API 计费 (BYOK)
GLM Plan 按模型分档计费（输入/输出分开），以官方价目为准
Cursor $20/月起, 含一定用量, 超出按量计费

自主度与控制

Claude Code 细粒度权限模式（default / acceptEdits / plan / dontAsk / bypassPermissions）
OpenCode 规则化审批机制（allow / ask / deny），可按命令与路径细化
GLM Plan 基础权限控制
Cursor 支持 Background Agent，适合并行执行与 IDE 协作

05 教学补充建议

为了让学习者“看完能上手、上手不翻车”，建议把对比课从静态参数扩展到实操与治理维度。

建议增加的教学模块

TEACHING UPGRADE
补充维度 为什么重要 课堂落地方式
同题 Benchmark 避免只看营销词，回到真实任务表现 统一任务：改接口 + 补测试 + 提 PR，对比成功率/耗时/人工介入
成本拆解 “价格”不等于“总成本” 拆成订阅费、Token费、失败重试、上下文长度与并发成本
权限与安全 决定是否能在团队/生产环境落地 演示 allow/ask/deny、危险命令拦截、敏感文件保护
生态可迁移性 降低未来迁移工具的重构成本 按 Rules / Skills / MCP / Hooks 设计“可移植提示资产”
失效数据治理 Stars/价格/模型能力更新很快 每页标注快照日期 + 官方链接，按月复核一次

选型建议

没有绝对最好的工具，只有最适合你的。建议先按你的真实任务（例如“改一个接口 + 补测试 + 提交 PR”）做同题试跑，再从成功率、耗时、人工介入次数与成本四个指标做选择：重度终端与深度 Agentic 可优先 Claude Code；追求模型灵活性和开源生态可优先 OpenCode；重视国内可达性与模型成本可尝试 GLM Plan；偏好 IDE 一体化与可视化流程可优先 Cursor。
