03 选型维度分析

从模型灵活性、成本控制、自主度三大维度深入分析，帮助你根据自身场景做出最优选择。

模型锁定 vs 灵活性

Claude Code 深度绑定 Claude，功能最先获得，但无法切换模型
OpenCode 任意模型提供商，最大灵活性，可用 Claude / GPT / Gemini / 本地模型
Gemini CLI 绑定 Gemini 系列，1M token 超长上下文，免费额度慷慨
Cursor 支持多模型切换，但部分高级功能仅限特定模型

价格与成本

Claude Code Pro $20/月, Max $100-200/月 (包含 Claude 用量)
OpenCode 工具免费, 按模型 API 计费 (BYOK)
Gemini CLI 免费层 60次/分 + 1000次/天；付费可通过 API Key 或 Vertex AI 按量计费
Cursor $20/月起, 含一定用量, 超出按量计费

自主度与控制

Claude Code 细粒度权限模式（default / acceptEdits / plan / dontAsk / bypassPermissions）
OpenCode 规则化审批机制（allow / ask / deny），可按命令与路径细化
Gemini CLI 工具审批机制 + YOLO 模式一键放行，Hooks 可自定义安全策略
Cursor 支持 Background Agent，适合并行执行与 IDE 协作

选型建议

没有绝对最好的工具，只有最适合你的。建议先按你的真实任务（例如"改一个接口 + 补测试 + 提交 PR"）做同题试跑，再从成功率、耗时、人工介入次数与成本四个指标做选择：重度终端与深度 Agentic 可优先 Claude Code；追求模型灵活性和开源生态可优先 OpenCode；看重免费额度、Google 生态与 1M 超长上下文可优先 Gemini CLI；偏好 IDE 一体化与可视化流程可优先 Cursor。
