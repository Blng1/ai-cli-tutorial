工作流程：一次完整的交互

WORKFLOW
claude "帮我修复 login 页面的表单验证 bug"
1
上下文组装
CLI 启动时自动加载项目级与用户级 CLAUDE.md、内置工具定义、已配置的 MCP Server，组装为 System Prompt 发送给 LLM。
CLAUDE.md Tool Definitions MCP Servers Conversation History
2
意图理解 & 步骤规划
LLM 解析用户指令，识别任务类型（Bug Fix），自主规划执行步骤——先定位代码，再分析问题，最后修复并验证。
LLM Reasoning
AGENTIC LOOP · 以下步骤可多轮迭代
3
代码定位
LLM 调用搜索工具，在代码库中定位与 login / validation 相关的文件和代码段。
Grep → "login.*valid" Glob → "src/**/login*" Hook: PreToolCall ✓
4
代码阅读 & 问题分析
LLM 调用 Read 读取目标文件，理解现有逻辑，定位 bug 根因——表单验证函数缺少邮箱格式校验。
Read → src/auth/login.ts Read → src/utils/validate.ts
5
代码修复
LLM 调用 Edit 工具，精确替换问题代码段。Hook 在写操作前拦截校验，确认符合项目规则后放行。
Edit → src/utils/validate.ts Hook: PreToolCall ✓ 校验写入
6
验证测试
LLM 调用 Bash 运行测试命令，确认修复有效且无回归。PostToolCall Hook 记录执行日志。
Bash → npm test --filter=login Hook: PostToolCall · 日志
测试失败则回到 Step 4 重新分析
7
结果输出
LLM 总结修复内容、改动范围和测试结果，以自然语言向用户报告。整个流程中用户只输入了一句话。
Output → 修复报告 Changed: 2 files
CONTEXT

Context Engineering

LLM 不是「什么都知道」——它只知道你喂给它的上下文。CLAUDE.md 定义项目规则与约束，Codebase Index 提供代码语义检索，Git History 补充变更脉络。上下文质量 = 输出质量。

决定性因素 CH.02
SKILLS

Skills 体系

将重复的工作流封装为可复用的 Skill。/commit 自动分析 diff 生成规范提交，/review 触发代码审查。Skills 让 AI 从「问答工具」升级为「自动化流水线」。

效率倍增 CH.03
TOOLS + MCP

工具调用与外部扩展

内置工具（Read/Write/Bash）覆盖本地操作；MCP 协议将能力边界扩展到外部——搜索引擎、知识库、DevOps 平台、自定义业务系统。MCP 是 AI CLI 的「USB 接口」。

内置 + 协议扩展 CH.04
HOOKS

Hooks 生命周期

在工具调用的前后插入自定义逻辑——执行前校验安全规则、执行后写入审计日志、状态变更时触发通知。Hooks 是你对 AI 行为的「守门人」，确保可控与合规。

安全 & 可控 CH.05

核心洞察：一切围绕 Context

架构图中的每个层级，本质上都在做同一件事——为 LLM 构建更好的上下文。CLAUDE.md 是静态上下文，Codebase Index 是动态上下文，Skills 是结构化上下文，MCP 是外部上下文，Hooks 是安全上下文。理解这一点，就抓住了 AI CLI 工具的核心。
